The Problem

Document Overload

Pharma teams receive a constant stream of clinical briefings, commercial memos, R&D protocols, and regulatory updates — mostly unstructured text with no clear owner.

Audience Mismatch

The same document means different things to Medical Affairs, Commercial, and R&D. A one-size summary serves no one well and wastes expert time.

Compliance Risk

Off-label promotion, absolute efficacy claims, safety minimisation — real regulatory and legal exposures buried in everyday content, invisible until it's too late.

Manual Triage

Without tooling, routing and summarisation is slow and inconsistent. Knowledge doesn't accumulate across teams — it evaporates with whoever read it.

Assort Design — What It Does

📥

Ingest

Paste text, submit a URL, or upload a file. Document stored and queued for processing immediately.

🎯

Route

TF-IDF + LR classifier picks the audience — Commercial, Medical Affairs, or R&D. LLM fallback for ambiguous cases.

✍️

Generate

Specialist agent writes a tailored summary, decision bullets, key clues, and mind map — audience-specific.

✅

Evaluate

Evaluator agent checks output quality. Fails? Pipeline revises automatically — up to 2 retries.

🔍

Validate

Citation Finder grounds claims to source text. Risk Checker scans for pharma compliance flags.

💾

Persist

All attempts, tags, citations, risk flags stored in SQLite — fully auditable and queryable.

Routing Intelligence

Mode A — ML Router (Primary)

Algorithm: TF-IDF vectorizer + Logistic Regression

Classes: commercial · medical_affairs · r_and_d

  • Top-class probability < threshold → cross-functional
  • Margin between top-2 < margin → cross-functional
  • Otherwise → top class wins
  • Returns top 5 TF-IDF signals per prediction

Mode B — LLM Fallback

When ML is uncertain, route_audience() calls the LLM and parses JSON with audience, confidence, candidates, reasons.

Auto-Retrain

Every RETRAIN_EVERY_N_JOBS (default 20) completed jobs, _maybe_retrain() fires automatically. No manual intervention required.

# graph.py — route_audience_node

ml_result = _ml_router.predict(
    document.content,
    threshold,   # e.g. 0.60
    margin       # e.g. 0.15
)

if ml_result["audience"] != "cross_functional":
    # ML confident — skip LLM entirely
    routing_source = "ml"
    audience   = ml_result["audience"]
    confidence = ml_result["confidence"]
    signals    = ml_result["top_signals"]
    # e.g. ["clinical", "patient", "safety"]
else:
    # ML uncertain → LLM fallback
    routing_source = "ml+llm_fallback"
    audience, confidence, ... = \
        _parse_llm_routing(...)

# Every decision persisted for audit trail
job.routing_source     = routing_source
job.routing_confidence = confidence
job.router_version     = ml_result["router_version"]

Explainability

explain(text) returns per-term TF-IDF × weight contributions for predicted class and runner-up — shown on the ML Router Insights page.

# ml_router.py — predict()

X     = vectorizer.transform([text])
proba = classifier.predict_proba(X)[0]

prob_map = {cls: proba[i]
            for i, cls in enumerate(cls_order)}

sorted_cls = sorted(prob_map.items(),
                    key=lambda kv: kv[1],
                    reverse=True)
p1_label, p1 = sorted_cls[0]  # top class
_,        p2 = sorted_cls[1]  # runner-up

# Two-level confidence gate
if p1 < threshold:
    audience = "cross_functional"
    reason = f"low confidence ({p1:.0%})"

elif (p1 - p2) < margin:
    audience = "cross_functional"
    reason = (f"ambiguous: {p1_label} {p1:.0%}"
              f" vs {sorted_cls[1][0]} {p2:.0%}")
else:
    audience = p1_label   # confident single pick

# Top 5 TF-IDF features for winning class
signals = _top_features(
    vectorizer, classifier,
    class_idx=cls_order.index(p1_label),
    n=5
)

Tools Layer — Citation Finder

What it does

For each generated decision bullet, citation_finder() locates the best-matching span in the source document — grounding AI output to the original text.

Algorithm (in order)

  1. Exact substring match → confidence 0.90
  2. Token-overlap: |overlap| / |claim_tokens|
  3. Score ≥ 0.25 → accepted (conf 0.40–0.85)
  4. Below 0.25 → ungrounded (conf 0.20)

Quality Gate Impact

If < 3 grounded citations and high-risk flags exist → pipeline hard-fails and forces a revision cycle.

# tools.py — citation_finder (simplified)

for claim in claims[:8]:
    claim_tokens = _tokenize(claim)

    # 1. Exact match — fastest path
    idx = source_text.lower().find(claim.lower())
    if idx != -1:
        results.append(CitationResult(
            confidence=0.90, ...))
        continue

    # 2. Token-overlap scoring
    best_score = 0.0
    for sentence, start, end in sentences:
        sent_tokens = _tokenize(sentence)
        overlap = claim_tokens & sent_tokens
        score = len(overlap) / len(claim_tokens)
        if score > best_score:
            best_score = score
            best = (sentence, start, end)

    # 3. Accept or mark ungrounded
    if best and best_score >= 0.25:
        results.append(CitationResult(
            confidence=min(0.85, 0.4+best_score),
            ...))
    else:
        results.append(CitationResult(
            confidence=0.20, quote_text=""))

Tools Layer — Pharma Risk Checker

Pure Python regex engine — no LLM, zero extra latency — scanning both source and generated bullets.

HIGH Severity

  • Absolute efficacy — "will cure", "100% efficacy"
  • Off-label promotion — "unapproved indication"
  • Safety minimisation — "minor side effects"
  • Boxed warning not prominently disclosed

MEDIUM Severity

  • Comparative — "best-in-class", "superior to"
  • Cross-trial comparison without head-to-head data
  • Patient testimonials as clinical evidence

LOW Severity

  • Missing limitation language (no "may/might/suggests")
  • Efficacy without safety context
  • Undisclosed KOL / conflict of interest
# tools.py — risk_checker (key rules)

# HIGH: off-label promotion
_off_label = re.compile(
    r"\b(off[\-\s]label"
    r"|unapproved\s+indication"
    r"|not\s+(yet\s+)?approved\s+for\s+\w+"
    r")\b", re.I)

# MEDIUM: comparative claims
_comparative = re.compile(
    r"\b(best[\-\s]in[\-\s]class"
    r"|superior(\s+to)?"
    r"|gold\s+standard"
    r"|only\s+(drug|therapy|treatment)"
    r")\b", re.I)

# LOW: missing limitation language
_caution = re.compile(
    r"\b(may|might|could|suggests?"
    r"|preliminary|potential"
    r")\b", re.I)

if not _caution.search(output):
    flags.append(RiskFlag(
        severity="low",
        category="missing limitation language",
        suggested_fix="Add qualifiers such as "

Tag Intelligence

Co-occurrence Pairs

Tags that appear together across documents — reveals what topics cluster in your pipeline across teams.

Jaccard Similarity

|A ∩ B| / |A ∪ B| — how similar two documents are by tag overlap. Finds functionally related content without reading it.

Bridge Tags

Tags that appear across multiple audience domains — cross-functional signals that matter to everyone simultaneously.

Tag Aliases

"PD-L1" and "PDL1" resolve to the same canonical tag. Editable via the UI — no code changes needed.

# tag_intel.py

def compute_cooccurrence_pairs(session):
    """Tags that appear together."""
    doc_tag_map = defaultdict(set)
    for dt in session.exec(select(DocumentTag)):
        doc_tag_map[dt.document_id].add(dt.tag_id)
    pairs = Counter()
    for tags in doc_tag_map.values():
        for a, b in combinations(sorted(tags), 2):
            pairs[(a, b)] += 1
    return pairs.most_common(20)


def compute_jaccard(tags_a: set,
                    tags_b: set) -> float:
    if not tags_a or not tags_b:
        return 0.0
    return len(tags_a & tags_b) / len(tags_a | tags_b)


def compute_bridge_tags(session):
    """Tags appearing in 2+ audience domains."""
    domain_tags = defaultdict(set)
    for s in session.exec(
            select(DocumentTagSummary)):
        tags = json.loads(s.canonical_tags_json)
        for t in tags:
            domain_tags[t].add(s.domain)
    return {t: ds
            for t, ds in domain_tags.items()
            if len(ds) >= 2}

The Self-Improving Loop

How it works

  1. Every completed job with resolved audience = a labeled training example
  2. After N new completions, _maybe_retrain() fires automatically
  3. train_model() re-fits TF-IDF + LR on the full corpus
  4. New .pkl artifacts written; _ml_router.reload() hot-swaps in memory

The feedback signal

LLM routing decisions act as weak labels for ML training. As confident decisions accumulate, ML takes over — reducing LLM calls, latency, and cost over time.

Thread-safe design

A threading.Lock ensures concurrent requests don't trigger parallel retrains. Non-fatal — app continues with previous model if training fails.

# graph.py — _maybe_retrain()

_RETRAIN_EVERY_N_JOBS = int(
    os.getenv("RETRAIN_EVERY_N_JOBS", "20"))
_retrain_lock = threading.Lock()

def _maybe_retrain(session: Session) -> None:
    completed = session.exec(
        select(Job)
        .where(Job.status == "completed")
        .where(Job.audience.in_([
            "commercial",
            "medical_affairs",
            "r_and_d"
        ]))
    ).all()
    n_completed = len(completed)

    last_n = json.load(metadata_file)\
                 .get("n_docs", 0)
    new_since = n_completed - last_n

    if new_since < _RETRAIN_EVERY_N_JOBS:
        return   # not enough new data yet

    # One retrain at a time — non-blocking
    if not _retrain_lock.acquire(blocking=False):
        return

    try:
        result = train_model(min_samples=10)
        if result:
            _ml_router.reload()  # hot-swap
    finally:
        _retrain_lock.release()

  • Prompt: clinical evidence, safety profile, KOL strategy

  • Prompt: mechanism of action, methods, pipeline implications
  • Required sections: Scientific Rationale · Methods · Pipeline Implications
  • Default max: 600 words
